## 功能说明

**可待**是一款基于 Python Flask 的**自主放号取号系统**，灵感来源于**医院叫号系统**，为了解决其他行业同样的放号及取号需求。**可待**同时具备前后端和数据库，是自主搭建的轻量化首选叫号系统平台，分权限管理，方便易用。

**可待**的用户分为三类，对应三种不同的身份和管理权限，分别是**项目管理员**、**放号员**、和**取号员**。

| 用户身份       | 用户权限                                                     | 简介                                                         | 应用举例                                                     |
| -------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **项目管理员** | 创建和管理项目、创建和管理活动、创建放号员账号、查看所有放号和取号信息。 | 项目的最高管理员，权限仅次于后台，可以拥有一切管理权限。     | 系统管理员、系统负责人、项目负责人等。                       |
| **放号员**     | 创建放号条目、查看本人放号和取号信息。                       | 项目的参与者，受管理员管辖，可以在自己参与的活动中放号。     | 在项目中需要放号的人员，如医院挂号系统的医生、一对一辅导的教师等。 |
| **取号员**     | 取号、删除已取号。                                           | 活动的参与者，无需登录，只能根据活动码进行取号，无其他权限。 | 在活动中需要取号的人员，如医院挂号系统的患者、一对一辅导的学生等。 |

**可待**中有三个概念需要区分，分别是**项目**、**活动**和**放取号条目**。

| 名词概念     | 参与者                     | 码号位数 | 简介                                                         | 应用举例                                     |
| ------------ | -------------------------- | -------- | ------------------------------------------------------------ | -------------------------------------------- |
| **项目**     | 项目管理员                 | 6位      | 项目指一系列活动总合，也可以指一系列项目的主办方。同项目可以包含多个活动，统一管理所有活动的放号员、填写条目、数据等。 | 北京XX医院挂号项目，XX学院朋辈辅导项目。     |
| **活动**     | 项目管理员、放号员、取号员 | 8位      | 活动指单次执行的活动，有一定的时间限制，常为周或日级的短期活动。放号员可以在活动中放号，取号员可以在活动中取号，项目管理员可以查看所有数据。 | 北京XX医院第XX周挂号，XX学院第XX周活动预约。 |
| **取号条目** | 项目管理员、放号员、取号员 | 10位     | 取号条目指取号员取号时的取号数据，包括放号员和取号员填写的内容、取号码等固定数据。 | XX挂号条目，XX活动预约号码条目。             |

## 部署

⚠️ **注意事项**
请确保在运行指令前已安装所有依赖 (`pip install -r requirements.txt`)。
如果使用了虚拟环境，请确保先激活虚拟环境。

**1 初始化/创建管理员**

首次部署或需要新增超级管理员时使用。

```bash
flask create-admin
```
作用：按照提示输入用户名和密码，创建一个拥有最高权限的“项目管理员”账号。

**2 启动系统**

```bash
flask run
# 或
python app.py
```

**3 数据库无损升级**

当更新了系统代码（增加了新功能或字段）但不想删除旧数据时使用。

```bash
flask update-db
```
作用：自动检测并添加缺少的数据库表和字段，同时为旧数据补全必要信息（如为旧预约生成取号码），不会丢失现有数据。

**4 查看所有用户**

查看当前系统中所有的管理员和放号员列表。

```bash
flask list-users
```
作用：以表格形式打印出所有用户的 ID、用户名、角色（管理员/放号员）以及所属项目 ID。

**5 修改用户账号/密码**

忘记密码或需要强制修改某人账号时使用。

```bash
flask edit-user
```
作用：交互式指令。先输入要修改的用户 ID（可通过 `list-users` 查看），然后按照提示输入新的用户名或密码。直接按回车键则保持原样不修改。
